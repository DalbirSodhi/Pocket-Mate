# Pocket-Mate Design Direction

## Design Goal

Pocket-Mate should feel calm, useful, and trustworthy. It should help users make quick daily money decisions without feeling judged or overwhelmed.

## Product Personality

- Clear.
- Practical.
- Friendly.
- Private.
- Calm.

## Visual Direction

- Light mode first.
- Neutral app background.
- White content surfaces.
- Clear financial numbers.
- Minimal chart usage.
- Strong color meaning.

## Suggested Palette

```text
Primary: deep green or confident blue
Savings: green
Warning: amber
Danger: red
Background: off-white or light gray
Text: near-black
Cards: white
```

## Main Screen Expectations

### Dashboard

The dashboard should immediately show:

- Safe-to-spend today.
- Income this cycle.
- Spent this cycle.
- Savings protected.
- Days until next payday.
- Budget pressure.

### Add Expense

The add expense screen should be fast:

- Amount.
- Category.
- Date.
- Optional note.
- Save.

### Budgets

Budgets should show:

- Category cap.
- Spent so far.
- Remaining amount.
- Progress indicator.
- Warning when close to cap.

### Savings

Savings should show:

- Goal amount.
- Current saved amount.
- Target date.
- Progress.
- Suggested contribution.

## UX Principle

The app should prefer clear answers over complex charts.

The most important answer is:

```text
Can I safely spend today?
```
